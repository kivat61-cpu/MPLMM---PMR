# PMR + MPLMM API 知识有效性判断项目说明文档

---

## 1. 项目概述

### 学术背景

本项目是一个多模态深度学习研究项目，将原用于**音频-视觉双模态分类**的 PMR（Prototype-based Modality Regulation）训练框架改造为**API 知识有效性判断**任务。同时集成了 MPLMM（Multimodal Prompt Learning with Missing Modality）的多模态融合 Transformer 作为可选融合模块。

**数据来源**：Stack Overflow 帖子 + 技术教程。每条数据包含文本知识（自然语言描述）和代码知识（代码片段），以及对应的 API 标识（FQN — 完全限定名）。

**实际应用场景**：给定一个 API 名称和一段候选知识（文本+代码），判断该知识是否是对应 API 的有效知识。可用于 API 知识检索、代码搜索等下游任务。

### 任务定义

- **输入**：文本知识（SO/教程中的自然语言描述）+ 代码知识（SO/教程中的代码片段）+ API 标识（FQN，拼接到文本和代码之前）
- **输出**：二分类标签（0/1）
- **目标**：判断给定的（文本，代码）知识对其所属 API 是否为有效知识

### 核心方法

| 组件 | 来源 | 作用 |
|------|------|------|
| BERT (bert-base-uncased) | HuggingFace | 文本编码器 |
| CodeBERT (microsoft/codebert-base) | HuggingFace | 代码编码器 |
| PMR 梯度调制 | 本项目的核心创新 | 基于原型相似度动态平衡双模态学习速度 |
| MPLMM (PromptModel/MULTModel) | MPLMM 库 | 交叉注意力 Transformer 融合 + 缺失模态重建 |
| Sum/Concat/FiLM/Gated Fusion | 本项目的 fusion_modules | 基础融合方法（仅 textcode 模型） |

## 3. 模型架构

### 三种模型对比

| 特性 | TextCodeClassifier | PromptTextCodeClassifier | MULTTextCodeClassifier |
|------|-------------------|--------------------------|------------------------|
| CLI 参数 | `--model_type textcode` | `--model_type prompt_textcode` | `--model_type mult_textcode` |
| 文本编码 | BERT [CLS] → Linear(768,512) | BERT → Pool → Linear(768,768) | BERT → Pool → Linear(768,768) |
| 代码编码 | CodeBERT [CLS] → Linear(768,512) | CodeBERT → Pool → Linear(768,768) | CodeBERT → Pool → Linear(768,768) |
| 融合方式 | Sum/Concat/FiLM/Gated | MPLMM PromptModel | MPLMM MULTModel |
| 交叉注意力 | 无 | 有（8×8=64 attention） | 有（8×8=64 attention） |
| 缺失模态重建 | 无 | 有（Prompt 机制） | 无 |
| 输出 | (feat_t, feat_c, out, out_t, out_c) | 同左 | 同左 |

### TextCodeClassifier 详细结构

```
text → BERT(bert-base-uncased) → [CLS](768) → Linear(768, embed_dim) → t (512)
code → CodeBERT(codebert-base) → [CLS](768) → Linear(768, embed_dim) → c (512)
t, c → FusionModule(Sum/Concat/FiLM/Gated) → out (2)
t → classifier_t → out_t (2)
c → classifier_c → out_c (2)
```

### PromptTextCodeClassifier 详细结构

```
text → BERT → last_hidden_state [B,512,768] → Pool(num_tokens) → [B,N,768] → Linear(768,768) → t_seq
code → CodeBERT → last_hidden_state [B,512,768] → Pool(num_tokens) → [B,N,768] → Linear(768,768) → c_seq
t_seq, c_seq, missing_mod → PromptModel → (feat_t[B,768], feat_c[B,768], out[B,2])
feat_t → classifier_t → out_t (2)
feat_c → classifier_c → out_c (2)
```

**Pool 策略**：
- `num_tokens == 1`：取 `[CLS]` token（和原始 BERT 逻辑一致）
- `num_tokens > 1`：`AdaptiveAvgPool1d` 将 512 个 token 压缩为 N 个等距 token

### PromptModel 内部流程（MPLMM）

```
输入: x_l [B, N, 768], x_a [B, N, 768], missing_mode [B]

1. 根据 missing_mode 选择路径：
   - mode=0 (缺文本)：用代码生成文本 → l_ap MLP 融合
   - mode=1 (缺代码)：用文本生成代码 → a_lp MLP 融合
   - mode=2 (完整)：直接 proj 投影 + prompt_nm

2. 交叉注意力 Transformer：
   - trans_l_with_a: 文本(Q) 关注 代码(K,V)
   - trans_a_with_l: 代码(Q) 关注 文本(K,V)
   - 拼接 batch_prompt

3. 自注意力记忆 Transformer：
   - trans_l_mem: 文本自注意力
   - trans_a_mem: 代码自注意力

4. 残差 MLP 输出层 → out [B, 2]
```

## 4. PMR 调制机制

### 五种调制策略

| 策略 | CLI 参数 | 原理 |
|------|----------|------|
| **Normal** | `--modulation Normal` | 标准训练，不做任何梯度调制 |
| **OGM** | `--modulation OGM` | On-the-fly Gradient Modulation：根据单模态验证 loss 动态缩放梯度 |
| **OGM_GE** | `--modulation OGM_GE` | OGM + Gradient Equilibrium：在 OGM 基础上叠加梯度均衡项 |
| **Acc** | `--modulation Acc` | 准确率调制：根据单模态准确率差异调制梯度 |
| **Proto** | `--modulation Proto` | 原型调制：计算每个模态到类别原型的欧氏距离，距离小的模态抑制、距离大的增强 |

### Proto（原型调制）详细流程

```
每个 epoch：
1. calculate_prototype()：用验证集前 10% 数据计算 text_proto[2, 512] 和 code_proto[2, 512]
   - 类别 c 的 text_proto[c] = 该类所有样本的 text 特征均值
   - 动量更新：new_proto = (1-0.2)*old_proto + 0.2*current_proto

2. 每个 batch：
   - text_sim = -EU_dist(t, text_proto)  # 负欧氏距离 = 相似度
   - code_sim = -EU_dist(c, code_proto)
   - ratio_t_p = softmax(text_sim)[label] / softmax(code_sim)[label]
   
   - ratio_t_p > 1：text 学得更好 → 抑制 text 梯度，增强 code 梯度
   - ratio_t_p < 1：code 学得更好 → 抑制 code 梯度，增强 text 梯度
   - loss = CE(out, label) + beta*loss_proto_t + lam*loss_proto_c
```

### Alpha 参数

`--alpha` 控制调制强度。alpha 越大，对学习领先模态的惩罚越重。建议：
- Proto 模式：alpha = 0.3~1.0
- OGM/OGM_GE 模式：alpha = 0.1~0.5

## 6. 数据流与训练流程

### 完整训练链路

```
┌─────────────────────────────────────────────────────────────────────┐
│ 1. main() → get_arguments() → 解析 CLI 参数                         │
├─────────────────────────────────────────────────────────────────────┤
│ 2. 如果是测试模式：                                                  │
│    - torch.load(ckpt_path) 读取 num_tokens                          │
│    - 如果与 CLI 不同，覆盖 args.num_tokens                           │
├─────────────────────────────────────────────────────────────────────┤
│ 3. 构建模型：                                                        │
│    - MPLMMConfig(args) 设置 hyp_params（含 seq_len = num_tokens）    │
│    - 根据 model_type 实例化对应模型类                                 │
│    - model.to(device)                                                │
├─────────────────────────────────────────────────────────────────────┤
│ 4. 构建优化器（差异化学习率）：                                       │
│    - BERT/CodeBERT 主干: lr = 2e-5                                   │
│    - text_proj, code_proj, fusion/prompt/mult, classifier_t/c:      │
│      lr = args.learning_rate (默认 1e-4)                             │
│    - 学习率 Warmup（前 3 epoch 线性从 1% 升至 100%）                  │
├─────────────────────────────────────────────────────────────────────┤
│ 5. 加载数据：                                                        │
│    - pd.read_csv("data/official/official_train.csv")                │
│    - TextCodeDataset(df, drop_rate, full_data)                      │
│    - DataLoader → 训练/验证/测试集                                   │
├─────────────────────────────────────────────────────────────────────┤
│ 6. 训练循环 (每个 epoch)：                                           │
│    a. calculate_prototype() → text_proto, code_proto                 │
│    b. train_epoch():                                                 │
│       - BERT/CodeBERT 编码 → Pool → 投影                             │
│       - MPLMM 交叉注意力融合 → out, out_t, out_c                      │
│       - 计算 CE Loss + Proto Loss                                   │
│       - PMR 梯度调制（如果 ratio_t_p != 1）                          │
│       - 梯度裁剪 (max_norm=1.0)                                      │
│       - optimizer.step()                                             │
│    c. valid() → 验证集评估                                           │
│    d. 保存 checkpoint（原子保存）                                     │
├─────────────────────────────────────────────────────────────────────┤
│ 7. 测试 (如果 --ckpt_path 指定)：                                    │
│    - 加载 state_dict（strict 模式）                                  │
│    - 从 checkpoint 加载 prototype                                    │
│    - valid() 在测试集上评估                                           │
│    - 输出：Accuracy, per-class acc, Precision, Recall, F1            │
└─────────────────────────────────────────────────────────────────────┘
```

### 数据加载流程

```
CSV 文件 → pd.read_csv() → DataFrame
  → TextCodeDataset.__getitem__(idx):
    1. 读取 row['text'], row['code'], row['label'], row['FQN']
    2. 判断真实缺失（NaN 或空字符串）
    3. 构建 API 前缀："[API: FQN] "（仅当 FQN 有效）
    4. 非缺失模态拼接 API 前缀，缺失模态保持 "[PAD]"
    5. BERT tokenizer 对 text 分词 → input_ids, attention_mask (max_length=512)
    6. CodeBERT tokenizer 对 code 分词 → input_ids, attention_mask (max_length=512)
    7. 决定 missing_mode:
       - 真实缺失 → missing_mode=0（缺文本）或 1（缺代码）
       - full_data=True → missing_mode=2（完整）
       - 否则 random() < drop_rate → 随机丢失一个模态
    8. 返回 dict: {text_input_ids, text_attention_mask, code_input_ids,
                   code_attention_mask, label, missing_mode}
  → DataLoader 打包为 batch
```

### 学习率策略

```
BERT (text_net)       : lr = 2e-5  ← 预训练权重，微小调整
CodeBERT (code_net)   : lr = 2e-5  ← 预训练权重，微小调整
text_proj / code_proj : lr = 1e-4  ← 随机初始化，正常学习
fusion/prompt/mult    : lr = 1e-4  ← 随机初始化，正常学习
classifier_t / _c     : lr = 1e-4  ← 随机初始化，正常学习

Warmup: 前 3 epoch，所有组的学习率线性从 0.01× 升至 1.0×
```

## 8. 关键参数调优建议

| 参数 | 建议范围 | 调优方向 |
|------|----------|----------|
| `--num_tokens` | 1, 4, 8, 16 | 数据量大选大值，数据集小选小值 |
| `--epoch` | 30~50 | 观察验证 loss 确定 |

### 已知问题

1. **旧 checkpoint 兼容性**：用 `num_tokens=1` 训练的 checkpoint 无法直接用于 `num_tokens=8`。已修复：加载时自动检测并覆盖 `--num_tokens`。
2. **旧 TextCodeClassifier checkpoint**：改造前保存的 checkpoint 只有 3 个返回值，不含 `classifier_t`/`classifier_c` 权重。需重新训练。
3. **hf-mirror.com 超时**：HuggingFace 镜像可能不稳定，设置 `export HF_ENDPOINT=https://huggingface.co` 切回官方源。

### 文件版本说明

- `main_Prototype_grad.py`：主训练脚本，支持 Proto 调制，三种模型
- `main_CE_test_grad.py`：备选脚本，仅支持 OGM/OGM_GE/Acc/Normal，仅基础 TextCodeClassifier
- `models/old_models/`：旧音频-视觉模型代码，不再使用
